# 日耕管理后台 — 角色权限矩阵

基于八大管理模块和四板块需求洞察文档中定义的角色体系，以下是各角色能看到的模块及操作权限。


## 一、角色定义

| 角色 | 对应人员 | 职责 |
|------|---------|------|
| **超级管理员** | 安老师、昊宇 | 系统全局管控、架构决策、最高权限 |
| **运营管理员** | 运营团队 | 平台日常运营、用户管理、内容审核 |
| **老师（顾问/教练）** | 签约入驻的老师 | 企业情报采集、学员辅导、项目协作 |
| **项目负责人** | 安老师/运营负责人 | 交付项目全局管控、老师分配监督 |
| **普通用户（学员）** | 产品使用者 | 无管理后台权限 |

## 三、各角色详细权限说明

### 3.1 超级管理员（安老师/昊宇）

**权限原则：** 全平台最高权限，所有模块可见、所有操作可执行。

| 模块 | 权限范围 | 特殊说明 |
|------|---------|---------|
| 用户与权限管理 | **全部操作** | 角色变更、VIP配置、贡献值规则、批量操作 |
| 教师管理 | **全部操作** | 入驻、授权、撤销、匹配干预 |
| 内容与知识库 | **全部操作** | 携君库增删改、安全规则、审核 |
| 系统运营监控 | **全部查看** | 含服务健康、AI Token、告警管理 |
| 消息推送管理 | **全部操作** | 规则配置、配额、公告 |
| 安全与危机监控 | **全部操作** | 危机事件、隐私审计、违规追踪 |
| 系统配置 | **全部操作** | 参数、开关、关联配置、问卷 |
| 审计日志 | **全部查看** | 含老师访问私有库审计 |
| 老师工作台 | **可模拟查看** | 可查看任意老师工作台状态 |

**特殊权限：**
- 可查看危机事件原始触发记录（脱敏后查看）
- 可配置系统级硬限制（推送频控、短信拦截）
- 可查看全部审计日志
- 可干预老师分配和项目监督


### 3.2 运营管理员（运营团队）

**权限原则：** 日常运营所需权限，负责用户运营、内容审核、数据监控。

| 模块 | 权限范围 | 限制说明 |
|------|---------|---------|
| 用户与权限管理 | **查看+部分操作** | 可查看全部用户；可打标签、批量导入导出；**不可**变更角色、配置VIP/贡献值规则 |
| 教师管理 | **查看+部分操作** | 可查看全部教师列表和工作量；**不可**入驻/授权/撤销老师 |
| 内容与知识库 | **查看+操作** | 可审核文档、管理携君库内容、审核用户贡献；**不可**配置安全规则 |
| 系统运营监控 | **只读** | 可查看仪表盘、存储、Token消耗、运营告警；**不可**查看服务健康状态 |
| 消息推送管理 | **查看+部分操作** | 可查看推送日志、发布系统公告；**不可**配置推送规则/配额 |
| 安全与危机监控 | **部分查看** | 可查看情绪异常用户预警（脱敏）、不活跃用户检测；**不可**查看危机事件日志、隐私审计 |
| 系统配置 | **无权限** | — |
| 审计日志 | **无权限** | — |
| 老师工作台 | **无权限** | — |

**特殊权限：**
- 可查看脱敏后的情绪异常用户列表，触发运营官微信私信关怀
- 可查看沉默用户列表，触发运营官微信私信关怀
- 可审核用户提交至公有库的内容（采纳/拒绝）
- 可发布系统公告


### 3.3 老师（顾问/教练）

**权限原则：** 仅限与自己相关的学员和项目，核心是辅导交付。

| 模块 | 权限范围 | 限制说明 |
|------|---------|---------|
| 用户与权限管理 | **受限查看** | 仅可查看分配给自己学员的基本信息和脱敏使用数据；**不可**查看其他用户、不可变更角色 |
| 教师管理 | **仅自己** | 可查看自己的工作量、绩效评分；**不可**查看其他老师、不可管理老师 |
| 内容与知识库 | **无权限** | — |
| 系统运营监控 | **无权限** | — |
| 消息推送管理 | **无权限** | — |
| 安全与危机监控 | **无权限** | — |
| 系统配置 | **无权限** | — |
| 审计日志 | **无权限** | — |
| **老师工作台** | **全部操作** | 专属工作台，详见下方 |

**老师工作台详细权限（第9模块）：**

| 功能 | 权限 | 说明 |
|------|:----:|------|
| 我的学员列表 | ✅ 查看+操作 | 仅分配给自己的学员，含基本信息、最近活跃 |
| 学员详情 | ✅ 查看 | 仅脱敏数据，含13模块使用概览 |
| 辅导预约管理 | ✅ 全部操作 | 查看待处理预约、确认/拒绝预约、发起辅导 |
| 内置视频辅导 | ✅ 全部操作 | 音视频通话+桌面共享+录屏+录音 |
| 企业情报采集 | ✅ 全部操作 | 独立入口，创建任务→AI采集→审核修正→交付用户 |
| 项目协作 | ✅ 查看+操作 | 仅自己参与的项目，可查看甘特图、确认节点、处理问题 |
| 学员动态监控 | ✅ 查看 | 学员在"拿下一个客户/打磨产品/交付订单"的关键进展 |
| 个人绩效概览 | ✅ 查看 | 仅自己的服务学员数、辅导时长、评分、成功率 |
| 保密协议签署 | ✅ 操作 | 签署/查看保密协议，未签署不可查看学员私有库 |
| 学员私有库访问 | ✅ 只读 | 经签署保密协议后，可查看分配学员的私有知识库（只读） |

**权限限制：**
- **绝对不可**查看未分配给自己学员的私有库
- **绝对不可**编辑/删除/导出学员私有库内容
- **绝对不可**查看学员情绪树洞原始倾诉内容（仅可查看脱敏后的情绪健康指数）
- 保密协议未签署时，私有库访问权限自动锁定


### 3.4 项目负责人（安老师/运营负责人）

**权限原则：** 交付项目全局管控，可监督和干预老师分配及项目进度。

| 模块 | 权限范围 | 限制说明 |
|------|---------|---------|
| 用户与权限管理 | **无权限** | — |
| 教师管理 | **部分操作** | 可查看老师列表；**可干预**用户-老师匹配关系；**不可**入驻/授权/撤销 |
| 内容与知识库 | **无权限** | — |
| 系统运营监控 | **受限查看** | 仅可查看**项目维度**的数据仪表盘（项目进度、逾期项目、交付率） |
| 消息推送管理 | **无权限** | — |
| 安全与危机监控 | **受限查看** | 仅可查看项目逾期相关告警 |
| 系统配置 | **无权限** | — |
| 审计日志 | **无权限** | — |
| 交付项目全局视图 | ✅ 全部 | 可查看全部交付项目、甘特图、问题追踪 |
| 老师工作台 | **可查看** | 可查看老师工作台状态（用于监督） |

**特殊权限：**
- 可干预用户-老师匹配（用户不满意可申请更换，由项目负责人处理）
- 可查看全部交付项目的甘特图和进度
- 可查看项目逾期超14天触发的运营关注告警
- 可监督老师服务质量


### 3.5 普通用户（学员）

**权限原则：** 无管理后台权限，仅使用客户端产品功能。

| 功能 | 权限 |
|------|:----:|
| 管理后台登录 | ❌ |
| 客户端13模块 | ✅ |
| 私有知识库管理 | ✅ |
| 个人数据查看 | ✅ |
| 查看老师工作台 | ❌ |

## 五、数据可见性层级（重要）

不同角色对同一数据有不同的可见粒度：

| 数据类型 | 超级管理员 | 运营管理员 | 老师 | 项目负责人 |
|---------|:----------:|:----------:|:----:|:----------:|
| 用户身份信息 | ✅ 全部可见 | ✅ 全部可见 | ⚠️ 仅自己学员 | ❌ |
| 用户使用行为数据 | ✅ 全部可见 | ✅ 全部可见（脱敏） | ⚠️ 仅自己学员（脱敏） | ❌ |
| 用户私有库内容 | ❌ 不可直接查看 | ❌ 不可查看 | ⚠️ 经保密协议+只读 | ❌ |
| 情绪树洞原始内容 | ❌ 绝对不可查看 | ❌ 绝对不可查看 | ❌ 绝对不可查看 | ❌ |
| 情绪健康指数 | ✅ 全部（聚合） | ✅ 全部（聚合脱敏） | ⚠️ 仅自己学员（聚合） | ❌ |
| 危机事件日志 | ✅ 脱敏查看 | ❌ | ❌ | ❌ |
| 携君库内容 | ✅ 全部 | ✅ 全部 | ❌ | ❌ |
| 项目数据 | ✅ 全部 | ✅ 全部 | ⚠️ 仅参与项目 | ✅ 全部 |
| 企业情报 | ❌ | ❌ | ✅ 自己采集的 | ❌ |

> **⚠️ 核心安全底线：**
> - **情绪树洞原始倾诉内容**——任何角色（含超级管理员）都不可查看
> - **用户私有库**——老师须签署保密协议后才可只读访问，且仅限自己学员
> - **老师访问私有库**——全量审计日志记录，超级管理员可追溯


## 六、权限变更与审批流程

| 变更类型 | 申请人 | 审批人 | 说明 |
|---------|--------|--------|------|
| 运营管理员新增/权限变更 | 运营负责人 | 超级管理员 | 需明确职责范围 |
| 老师入驻/授权 | 运营管理员 | 超级管理员 | 需签署保密协议 |
| 老师权限撤销 | 运营管理员/超级管理员 | 超级管理员 | 立即生效 |
| 用户→老师角色升级 | 用户申请/运营推荐 | 超级管理员 | 需审核资质 |
| 项目负责人权限授予 | 超级管理员 | 超级管理员 | 按项目分配 |
| 普通用户VIP升级 | 系统自动/用户付费 | — | 自动化流程 |
