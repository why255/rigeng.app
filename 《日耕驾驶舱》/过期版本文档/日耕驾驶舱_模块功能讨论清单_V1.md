## 模块一：品牌定位舱  `履历细细耕，IP自然成`

**目标**：通过对话式访谈帮用户搞清"我在HR圈是谁"，生成丰满的 IP 定位资产，为后续内容/分发/获客奠基。

| #    | 功能名             | 功能描述                                    | 关键实现要点 / 技术约束        | 标签    |
| ---- | --------------- | --------------------------------------- | -------------------- | ----- |
| 1.1  | 对话式 IP 访谈引擎     | 多轮自然语言交流替代填表，挖掘用户履历、故事、专长、差异化卖点         | LLM 多轮对话、上下文窗口、状态机管理 | 🔴 必须 |
| 1.2  | 智能实时分析引擎        | 实时分析用户回答，判断信息丰满度，决定是否需要追问               | 丰满度评分模型、追问触发规则       | 🔴 必须 |
| 1.3  | 智能追问生成器         | 基于当前画像缺口自动生成下一轮追问问题                     | Prompt 模板 + 缺口驱动     | 🔴 必须 |
| 1.4  | 后台话术与追问配置中心     | 管理员可配置问题、追问逻辑、顺序，灵活调整                   | 配置化对话流（低代码）、版本管理     | 🔴 必须 |
| 1.5  | 履历结构化解析         | 从用户回答中自动提取关键经历、成就、专业标签、个人故事线            | NER / 信息抽取 / 结构化输出   | 🔴 必须 |
| 1.6  | IP 定位算法与人设生成    | 识别专业领域 + 差异化卖点 + 人设风格（温润型/专家型/陪伴型/转型型等） | 标签体系、人设分类法           | 🔴 必须 |
| 1.7  | IP 丰满度评估与自动引导补充 | 评估 IP 立体度，不足时引导用户继续补充                   | 评估指标体系               | 🟡 重要 |
| 1.8  | IP 名片与个人简介生成器   | 自动生成图文版个人简介（适配公众号 / 视频号 / 小红书）          | 多模板、多平台规格            | 🟡 重要 |
| 1.9  | 个人故事模板库         | 成长故事 / 转型故事 / 专业观点故事模板，后台可配置            | 模板引擎 + 变量替换          | 🟡 重要 |
| 1.10 | 品牌调性校验引擎        | 在 IP 输出环节也调用全局品牌语校验                     | 复用全局横切能力             | 🔴 必须 |

> **讨论点**：多轮对话的状态机和上下文管理是 LLM 应用常见痛点，建议 技术负责人 重点评估 token 消耗与延迟。

## 模块二：内容采撷舱  `内外双通道，好料自然来`

**目标**：打通日耕私有知识库 + 外部公域爆款内容的双向通道，让用户"无感"获得高质量素材。

| #    | 功能名                        | 功能描述                                 | 关键实现要点 / 技术约束                   | 标签    |
| ---- | -------------------------- | ------------------------------------ | ------------------------------- | ----- |
| 2.1  | 日耕私有知识库 API                | 一次性授权，自动抓取用户在日耕 APP 的复盘、问答、方案        | 鉴权 OAuth、增量同步、防重                | 🔴 必须 |
| 2.2  | 后台博主对标配置中心                 | 管理员在后台配置对标博主列表，用户端零感知                | 配置面板、博主档案管理                     | 🔴 必须 |
| 2.3  | 多平台爬虫引擎                    | 支持抖音 / 视频号 / B站 / 小红书 / 公众号公开内容采集    | **强约束**：仅采公开可下载内容、反爬应对、内容版权风险评估 | 🔴 必须 |
| 2.4  | 博主内容自动监控                   | 自动监控已配置博主的最新内容更新                     | 轮询任务调度（默认 20 小时）、爆款实时监控         | 🟡 重要 |
| 2.5  | 智能筛选引擎                     | 按点赞 / 评论 / 转发 / 收藏 / 时间等多维筛选         | 排序规则可配置                         | 🔴 必须 |
| 2.6  | HR 模块智能分类                  | 自动归类到招聘 / 培训 / 薪酬 / 绩效 / 规划 / 员工关系 等 | 分类模型、HR 行业词库                    | 🔴 必须 |
| 2.7  | 视频 → 音频 → 文字 → Markdown 链路 | 自动分离音轨、ASR 转写、Markdown 化             | ASR 选型（HR 术语词库）、分段算法            | 🟡 重要 |
| 2.8  | 内容去重与质量评分                  | 去重 + 智能评分筛选高价值素材                     | 文本指纹、质量模型                       | 🟡 重要 |
| 2.9  | 素材库管理                      | 按分类 / 标签 / 来源 / 时间归档，支持搜索            | 全文检索（ES/Meilisearch）            | 🟡 重要 |
| 2.10 | 采集频率配置后台                   | 默认 20 小时轮询，爆款实时，可改                   | 调度框架、优先级队列                      | 🟡 重要 |

> **讨论点（高风险）**：多平台爬虫会涉及**反爬策略、内容版权合规与平台协议** —— 这是 技术负责人 必须给合规结论的红线功能，必要时考虑"接入官方 API 或合规数据源"做替代方案。

## 模块四：全网分发舱  `规则配一次，全网自动耕`

**目标**：多平台账号统一管理 + 自动发布 + 效果回收。

| #   | 功能名       | 功能描述                                    | 关键实现要点 / 技术约束                                   | 标签    |
| --- | --------- | --------------------------------------- | ----------------------------------------------- | ----- |
| 4.1 | 多平台账号授权管理 | 公众号 / 抖音 / 视频号 / 小红书 / B站 绑定（OAuth 或账密） | **风控**：账密方式风险高，优先官方 OAuth；视频号生态需个人微信 SCRM 中后期规划 | 🔴 必须 |
| 4.2 | 发布规则配置后台  | 时间 / 频率 / 平台-内容映射 / 自动 vs 手动            | 规则引擎                                            | 🔴 必须 |
| 4.3 | 内容发布日历    | 可视化排期，拖拽调整                              | 前端日历组件                                          | 🟡 重要 |
| 4.4 | 自动发布引擎    | 图文 / 视频 / 音频自动发布到各平台                    | **稳定优先**：异步任务队列、重试、幂等、限流                        | 🔴 必须 |
| 4.5 | 发布状态实时监控  | 成功 / 失败 / 审核中提醒，失败自动重试                  | 状态机 + 重试策略                                      | 🔴 必须 |
| 4.6 | 效果数据追踪    | 阅读 / 点赞 / 评论 / 转发 / 收藏 / 粉丝增长           | 各平台开放接口 + 定时回传                                  | 🟡 重要 |

| 4.8 | 多维度数据报表   | 日 / 周 / 月报表                             | 报表引擎                                            | 🟡 重要 |

> **讨论点**：平台官方开放接口的能力/限额差异很大，特别是视频号，**完全自动化发布可能受限**，建议与各平台官方支持确认可行性后再定方案。

## 给讨论会的三个核心议题（建议开场先定调）

1. **风控与合规边界**：精准获客舱的"自动评论/私信/加好友"、私域耕耘舱的"聊天记录读取与自动回复"，这两个能力在第三方平台协议和账号安全上是高风险 —— 决定本期是"全自动"还是"半自动 + 人工确认"。
2. **数字人质量底线**：2K 分辨率 + 口型同步 + 表情自然，开源方案不达标时是否预留自研或采购预算 —— 决定内容智造舱能否按期上线。
3. **日耕APP积分打通**：打通的 API 契约、幂等、对账机制必须在 Phase 1 提前定义 —— 否则下游成交、结算模块都会受阻。
