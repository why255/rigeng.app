## 二、问题 1：git checkout HEAD 恢复旧版文件

### 现象

用户要求"恢复朝有规划和暮有复盘代码"时，执行了：

```bash
git checkout HEAD -- frontend/src/pages/board1/EveningReview*.tsx
```

这恢复了 HEAD 中**已被重构删除**的 4 个旧文件（全 emoji 图标），覆盖了 `evening-review/` 目录下的重构版本正确架构。

### 根因

`d4ef029` commit 已将 EveningReview 从 `board1/`（4 独立路由）重构到 `evening-review/`（Layout + 嵌套路由）。旧文件在 HEAD 中仍然存在（未被 git 标记为删除），`git checkout HEAD` 直接将它们恢复到 working tree，导致：

1. App.tsx 的 import 指向了旧 `board1/` 路径
2. 路由从 Layout+嵌套回退到 4 独立路由
3. 所有 emoji 回归

### 影响范围

- `EveningReviewEntry/Chat/Report/History` ×4 文件
- `App.tsx` 的 import 和路由配置

### 修复方式

```bash
# 1. 删除错误恢复的旧文件
rm frontend/src/pages/board1/EveningReview*.tsx

# 2. 恢复 App.tsx 到 evening-review/ 架构
# import: @/pages/board1/EveningReview* → @/pages/evening-review/*
# route: 4独立 → Layout+嵌套

# 3. 全局 emoji→Icon 替换
```

### 教训

执行 `git checkout/restore` 前必须三步卡控：

```
1. git status -- <path>      → 确认是 deleted / modified
2. git log --oneline -3 -- <path>  → 确认是否被有意重构
3. ls <替代目录>               → 检查是否有新目录替代
```

**关键信号**：`git status` 显示 `deleted` + `git log` 显示重构类 commit → **绝不恢复旧版**。

## 五、问题 4：.mh-page max-width 被错误移除

### 现象

用户反馈"情绪树洞布局大小不对，页面被改坏了"。

### 根因

在"规范化清理"阶段，把 `.mh-page { max-width: 680px }` 改成了 `max-width: none`，导致内容区域撑满全屏。

### 原型约束

全部 4 个 HTML 原型文件都使用 `max-w-[680px]`：

| 原型 | 行号 | 代码 |
|------|------|------|
| m3p1.html | 182 | `<div class="max-w-[680px] mx-auto space-y-8">` |
| m3p2.html | 151 | `<div class="max-w-[680px] mx-auto space-y-8 flex flex-col items-center justify-center w-full">` |
| m3p3.html | 178 | `<div class="max-w-[680px] mx-auto space-y-8">` |
| m3p4.html | 224 | `<div class="max-w-[680px] mx-auto space-y-6">` |

### 修复

```css
/* mood-haven.css */
.mh-page {
  max-width: 680px;  /* 恢复为原型设计值 */
  margin: 0 auto;
  padding: 48px 16px;
  width: 100%;
}
```

## 六、问题 5：图标与原型不一致

### 现象

两处 Icon 名称与 HTML 原型不匹配：

| 位置 | 原型 (m3p1/m3p4) | 错误代码 |
|------|-----------------|---------|
| P1 安全承诺头像 | `mingcute:face-line` | `mingcute:leaf-line` |
| P4 趋势图卡片头 | `mingcute:chart-line-line` | `mingcute:chart-bar-line` |

### 根因

接入指南使用了 `leaf-line` 作为树洞模块图标，但原型设计用 `face-line` 表达"小耕"（人物形象）。chart-line-line 才是原型中趋势图使用的图标。

对照原型逐行核对后纠正。
