## 〇、模块定位与价值

### 一句话定义

**通过对话式访谈，帮 HR 经理人在 30 分钟内搞清楚"我在 HR 圈是谁"，自动生成丰满的 IP 定位资产，为后续内容生产、私域耕耘、成交转化奠定基调。**

### 在八大模块中的位置

```
[ M1 品牌定位舱 ] → [ M2 内容采撷舱 ] → [ M3 内容智造舱 ] → ...
       ↑
   "我是谁"          "找好料"            "出精品"
   全链路起点         依赖 IP 调性        依赖 IP 调性
```

### 核心价值

- **耕愈工作**：搞清楚"我在 HR 圈是谁"，让专业被看见，建立职场辨识度
- **耕暖生活**：个人品牌是变现的根基，品牌立住了，价值才能自然来
- **底座价值**：后续 7 个模块的内容调性、人设风格、朋友圈文案、群发话术、产品推荐话术，全部依赖 M1 输出的 IP 画像

## 三、用户交互流程（User Flows）

### 3.1 主流程：首次 IP 访谈（S1 + S2 + S3 完整版）

```
[开始] 7天激活 Day 1 / 用户主动进入
   ↓
[Step 1] 欢迎页：小耕出现
   - 文案："我是小耕，30 分钟帮你把 HR 圈的人设立起来"
   - 两个按钮：[ 开始访谈 ] [ 之前中断过，继续对话 ]
   ↓
[Step 2] 暖场对话（3-5 轮）
   - 系统：聊聊你在 HR 这行做了多久？
   - 用户：8年
   - 系统：那你最常打交道的是哪类人？老板、HR同行、还是候选人？
   - ... 暖场，不计入正式信息
   ↓
[Step 3] 6 大维度轮番深挖（可乱序、按丰满度动态调整）
   ├─ 维度A：履历线（年限、职级、关键公司、行业）
   ├─ 维度B：专长线（擅长模块：招聘/培训/薪酬/绩效/规划/员工关系）
   ├─ 维度C：战绩线（做成过什么，有数据更好）
   ├─ 维度D：故事线（成长故事、转型故事、印象深刻的项目）
   ├─ 维度E：差异化（与同行的不同、独特视角）
   └─ 维度F：人设风格（温润型/专家型/陪伴型/转型型，系统给 3 个选项让用户选）
   ↓
[Step 4] 智能追问（实时）
   - 用户回答 → 智能分析 → 丰满度评分 → 不够则追问
   - 追问示例："你说擅长招聘，能不能讲一个最让你有成就感的招聘案例？"
   - 追问规则：每个维度至少 3 个信息点，丰满度 < 阈值时持续追问
   ↓
[Step 5] 实时进度反馈
   - 右上角进度环：履历 80% / 专长 60% / 战绩 40% / 故事 30% / 差异化 20% / 人设 100%
   - 全部 ≥ 阈值 → 进入 Step 6
   ↓
[Step 6] IP 定位报告生成（5-10 秒）
   - 系统提示："小耕正在给你做画像..."
   - 完成后展示：
     ├─ 人设标签（3-5 个）：如"实战派 HRD / 组织发展教练 / 招聘老兵"
     ├─ 专业领域：招聘体系搭建 + 组织诊断
     ├─ 差异化卖点："既懂老板视角，又懂候选人温度"
     ├─ 人设风格：温润型
     ├─ 一句话定位："8年招聘老兵，帮中型企业 3 个月内找到对的人"
     └─ 品牌故事线开头："从 500 人的校招独苗，到 5000 人 HRD 的十年修行"
   ↓
[Step 7] 用户确认 / 修改
   - 三个按钮：[ 都对，生成资产 ] [ 改一改 ] [ 重新访谈 ]
   - "改一改"支持逐项修改，系统记录为铁律
   - "重新访谈"重置对话，保留历史版本
   ↓
[Step 8] IP 资产生成（1-3 秒）
   ├─ IP 名片（公众号版 / 视频号版 / 小红书版 / 朋友圈版）
   ├─ 个人简介（三种长度：50字 / 150字 / 300字）
   ├─ 个人故事（V1.0 出成长 + 观点，V1.1 加转型）
   ├─ 专业背书话术（5 句）
   └─ 调性关键词（10 个）
   ↓
[Step 9] 一键应用 + 后续引导
   - "这张 IP 画像已经准备好，接下来[ 一键去采集内容 ] [ 看看我能发什么 ]"
   - 进入 M2 内容采撷舱
   ↓
[结束]
```

### 3.2 中断恢复流程（S2）

```
[用户随时退出] → 系统自动保存对话状态
   ↓
[用户再次进入] → Step 1 显示"继续上次对话"
   ↓
[用户点击继续] → 直接回到上次最后一步
   - 显示对话历史（可滚动查看）
   - 右上角进度环显示上次状态
   - 继续追问
```

### 3.3 IP 迭代流程（S4）

```
[用户已生成 IP] → 主动进入品牌定位舱
   ↓
[显示当前 IP 画像] + 三个按钮
   ├─ [ 更新信息 ]：补充新情况（升职/换行业/新案例）
   ├─ [ 重新定位 ]：完全重做（保留历史版本可查看）
   └─ [ 查看历史 ]：看历次版本 + 切换"启用版本"
```

### 3.4 老师查看学员 IP 流程（S5）

```
[老师进入工作台] → 选择学员 → 进入学员详情页
   ↓
[学员详情页] 多个 tab：IP 档案 / 阶段点评 / 成长数据
   ↓
[点击 IP 档案] → 看到学员当前 IP 画像（只读）
   - 不可修改，可加备注"亮点 / 待提升"
   - 备注沉淀到阶段点评
```

## 五、核心功能详细说明

### 5.1 对话式 IP 访谈引擎（🔴 P0）

**功能目标**：用自然对话替代填表，30 分钟内完成 6 大维度信息采集。

**核心机制**：
- **状态机**：`idle → intro_round → asking → awaiting_answer → analyzing → asking(loop) → ready_for_synthesis`
- **上下文管理**：单 session 累加 token 上限 16k；超出触发"对话压缩"（保留关键句 + LLM 摘要）
- **暖场环节**：3-5 轮轻松对话降低用户紧张感，不计入正式信息
- **维度调度**：6 维度按"丰满度最低优先"动态调度，不是固定顺序
- **节奏控制**：每轮回复限制 500 字内，系统根据回复长度动态调整追问颗粒度
- **跨端持久**：用户从 APP 切到 Web，对话进度无缝衔接（基于云端 sessionId）

**技术约束**：
- 调用智能统一接入层（Phase 1 组件）
- LLM 选型必须支持中文长上下文 + 角色一致性
- 单次访谈 token 消耗预估：8K-15K（单用户单次）
- 对话响应延迟：LLM 推理 < 8s，首字 < 2s（流式输出），单轮 P95 ≤ 4s
- 对话状态实时持久化（Redis + DB 双写）
- 退出后 30 天内可继续
- 对话可最少持续 4 轮，最多 30 轮；30 轮未达丰满度阈值也允许收网

**异常处理**：
- 用户回复太短（如"嗯"、"还行"）：智能追问补问"能再具体说说吗？"
- 用户回复离题：温和拉回"我注意到你提到了 XX，这个挺有意思的，能展开讲讲吗？"
- 用户情绪烦躁：智能降低追问频率，提示"咱们可以先停一下，下次再聊"
- LLM 调用失败：自动回到"已缓存问题模板"的第 N 题，保证对话不中断

### 5.2 智能实时分析引擎（🔴 P0）

**功能目标**：每轮用户回答后，实时评估信息丰满度，决定下一步动作。

**输入**：当前对话全量上下文 + 已抽取的"履历要素"（专业领域、履历年限、典型成就、个人故事等）维度是否填充。

**输出**：
- 丰满度评分（0~100）
- 缺口列表（"用户没讲过转型故事" / "差异化卖点还没说出来"）
- 决策信号：`need_more` / `ready_to_synthesize`

**核心机制**：
- 6 维度丰满度评分：每个维度 0-100 分
- 评分维度：
  - 关键信息点数量（履历：职级/年限/公司）
  - 信息具体度（数据/案例/场景）
  - 情感丰度（故事感、画面感）
  - 差异化程度（是否与同行模板化回答雷同）

**关键算法**（V1.0 权重）：
- 履历要素覆盖度（结构化字段是否填齐）：权重 60%
- 故事性覆盖（个人故事 / 转型 / 价值观三类至少出现两类）：权重 25%
- 表述独特性（避免和"通用 HR"模板重合度 ≥ 70%）：权重 15%

**调度策略**：
- 丰满度 < 30：必须追问
- 30 ≤ 丰满度 < 70：可选追问
- 丰满度 ≥ 70：跳过，进入下一维度

**关键实现要点**：
- 评分模型 V1.0 用规则 + LLM-as-judge 双轨，置信度 ≥ 0.8 才允许触发 `ready_to_synthesize`
- LLM 评分 + 规则评分双轨，加权平均
- 阈值后台可配（丰满度阈值、追问开关）

### 5.3 智能追问生成器（🔴 P0）

**功能目标**：根据当前画像缺口，自动生成下一轮追问问题。

**输入**：
- 当前丰满度缺口列表（来自 1.2）
- 用户最近 2 轮回答（避免重复）
- 后台话术库（来自 1.4）

**输出**：下一轮问题文本（≤ 50 字，温润调性，符合品牌语校验）

**核心机制**：
- **缺口驱动**：先识别"哪个维度丰满度最低"，再生成针对该维度的追问
- **追问模板库**：后台配置追问模板（每个维度至少 5 个）
- **Prompt 动态注入**"当前已知信息"，让追问具体化

**追问质量标准**：
- 不重复已问过的问题
- 语气符合日耕调性（温润、像朋友聊天）
- 不超过 50 字
- 给用户回答留空间（不诱导式提问）
- 必须落到 1 个具体事件或场景（避免"你有什么优点"这种空话）
- 同一缺口最多连续追问 3 轮，3 轮未给出即换缺口
- 必带"职业病式话术"作为反面教材（"赋能" → "陪伴"）

**技术约束**：
- 追问模板版本管理（后台可回滚）
- 追问必须经过品牌语校验（对外文案规范）

### 5.4 后台话术与追问配置中心（🔴 P0）

**功能目标**：管理员可在后台灵活配置对话流程，无需发版。

**核心机制**：
- **5 套维度配置**：开场白 / 履历追问 / 故事追问 / 风格确认 / 收尾
- **4 种节点类型**：
  - `问题节点`：开场问题
  - `追问节点`：基于丰满度触发
  - `摘要节点`：对话压缩时的提示语
  - `跳转节点`：条件跳转（如"如果用户已离职，跳到创业故事线"）
- **可视化编辑**：拖拽式流程编辑，所见即所得
- **问题模板变量化**：`{{USER_NAME}}` `{{INDUSTRY}}` `{{ROLE_LEVEL}}` 由系统填充
- **版本管理**：每次保存生成版本号，支持回滚
- **测试沙箱**：配置完可在沙箱里跑一遍完整对话
- **A/B 实验**：同一问题支持 N 个候选话术，按用户标签分流
- **灰度生效**：可指定用户群体试运行；配置修改后 5s 内线上生效

**技术约束**：
- 节点配置存储为 JSON Schema
- 节点间跳转用表达式（如 `{{score.履历}} < 30`）
- 仅"运营管理员"角色可写；"老师"角色只读

### 5.5 履历结构化解析（🔴 P0）

**功能目标**：从用户对话中自动提取结构化履历信息，喂给 1.6（人设生成）。

**输入**：对话全文。

**输出**（结构化 JSON）：
```json
{
  "industry": "制造业",
  "role_track": "薪酬绩效",
  "years_experience": 8,
  "core_achievements": [
    "主导 800 人规模薪酬体系重构",
    "HRSSC 筹建 0→1"
  ],
  "personal_story": [
    { "type": "转型", "summary": "从工程师转行做 HR" },
    { "type": "里程碑", "summary": "..." }
  ],
  "differentiation": "擅长把薪酬数据翻译成老板听得懂的故事",
  "tone_preference": "温润",
  "expertise_tags": ["薪酬体系", "数据化 HR", "中大型制造业"]
}
```

**核心机制**：
- **NER（命名实体识别）**：公司名、职位、年限、项目、数据
- **关系抽取**：职位-公司、年份-职位、案例-数据
- **信息归一化**：同义公司名合并（腾讯/腾讯科技）、职位标准化（HRM/HR经理 → HR 经理）
- **HR 行业词库**：内置 HR 模块词库（招聘/培训/薪酬/绩效/规划/员工关系），自动归类
- 抽取失败字段标 `null`，由 1.7 丰满度评估器知道这是缺口
- 输出合规：所有结构化字段都进品牌语校验（1.10）

**验收指标**：100 条样本对话，结构化字段抽取准确率 ≥ 85%。

### 5.6 IP 定位算法与人设生成（🔴 P0）

**功能目标**：基于结构化履历，生成人设标签、专业领域、差异化卖点、人设风格。

**人设分类（V1.0 定义 4 类，后台可扩展）**：

| 类型 | 调性 | 适配场景 |
|------|------|---------|
| 温润型 | 治愈 / 陪伴 / 走心 | 心理 / 员工关系 / 培训 |
| 专家型 | 数据 / 体系 / 权威 | 薪酬 / 绩效 / 组织发展 |
| 陪伴型 | 共情 / 倾听 / 长期 | 教练 / 顾问 / 长期合作 |
| 转型型 | 突破 / 重启 / 跨界 | 转行 / 跨行业 / 二次出发 |

**核心机制**：
- **人设判定**：LLM 评分每个类型的匹配度，取 Top 3 让用户选择
- **差异化卖点提炼**：从履历 + 故事中提取"独特组合"（如"500 强背景 + 创业公司经历"= 兼具体系感和落地感）
- **一句话定位公式**："[年限]+[身份]+[核心专长]+[对谁说]+[带来什么价值]"
- 分类法后台可配（运营管理员可调权重），预留 8-10 种扩展开关
- 人设分类命中率低时，1.6 多分类候选 + 用户投票选主用

**输出**：
- **人设标签**（1 个主标签 + 2 个副标签，如「温润型 + 专家型 + 陪伴型」）
- **一句话定位**（≤ 30 字，符合 10 字 5+5 句式品牌语调性，例"专注薪酬体系，陪你把数据翻译成价值"）
- **专业领域 + 差异化卖点**
- **人设风格**

**输出合规**：
- 输出必须经品牌语校验
- 一句话定位生成后必须过品牌语校验（1.10），不达标由 LLM 自检重试最多 2 次
- 用户可手动覆盖生成结果，覆盖行为进入「铁律」（与全局"用户偏好与铁律存储"对齐）

### 5.7 IP 丰满度评估与自动引导补充（🟡 P1，V1.0 基础版）

**功能目标**：把 V1.0 简版丰满度评估做到产品级：覆盖度 + 故事性 + 独特性 + 一致性。

**评估指标体系（V1.0）**：

| 维度 | 子项 | 满分 |
|------|------|------|
| 覆盖度 | 行业 / 角色 / 年限 / 核心成就 | 40 |
| 故事性 | 转型故事 / 里程碑故事 / 价值观故事 至少 2/3 | 30 |
| 独特性 | 与"通用 HR"模板重合度 | 15 |
| 一致性 | 各字段之间无矛盾（如"温润型"不匹配"擅长训斥"） | 15 |

**自动引导补充**：
- 总分 < 60 → "再聊 2 段就成形"，引导继续对话
- 总分 60~80 → 推荐 3 个强化方向
- 总分 > 80 → 直接进入名片生成（1.8）

**约束**：1.7 评估分数与 1.2 一致，不允许两套分数。

### 5.8 IP 名片与个人简介生成器（🟡 P1）

**功能目标**：基于 IP 资产，一键生成多平台规格的个人介绍。

**输出**：

| 平台 | 形态 | 长度上限 | 风格备注 |
|------|------|---------|---------|
| 微信公众号 | 长图文 + 头图 | 500 字 | 可读性强，可含数据案例 |
| 小红书 | 图文 | 300 字 + 9 图 | 生活化、有 emoji、有标签 |
| 视频号 | 60s 口播稿 | 180 字 | 钩子 + 简介 + CTA |
| 朋友圈（私域耕耘舱消费） | 短引文 | 50 字 | 千人千面，携带昵称 |

**关键实现要点**：
- 所有平台规格都经品牌语校验
- 头图 / 封面由智能配图引擎（3.20）生成
- 用户可手动微调，微调进入铁律
- 模板配置在后台，支持新增平台
- 配图走智能配图（全局组件，Phase 1）
- 智能数字人头像（Phase 1 组件，先静态头像，后期支持动态）

### 5.9 个人故事模板库（🟡 P1，V1.1 迭代）

> V1.0 不交付，本条仅占位，以便 Phase 3 接力。

后续交付：成长 / 转型 / 专业观点 / 价值观 四类模板；后台可配；用户选择后系统填充字段。

V1.0 资产页只生成"成长" + "观点" 两类基础故事。

### 5.10 品牌调性校验引擎（🔴 P0）

**功能目标**：本舱所有"对外文案"输出时调用全局校验。

**输入**：任意准备输出的字符串。

**输出**：
- 通过 / 不通过
- 不通过时：违规词位置 + 替换建议

**关键约束**：
- 必查禁用词（与全局术语规范对齐）：
  - `AI` → `智能`
  - `赚钱 / 变现 / 佣金 / 收益 / 分成` → `收获 / 价值 / 发光 / 被看见`
  - `推广 / 营销 / 拉新 / 收割` → `运营 / 耕耘 / 触达 / 陪伴`
  - `工具 / 系统 / 助手` → `驾驶舱 / 平台 / 陪伴`
- 必查句式：品牌语 10 字 5+5 调性（"日耕朝夕，耕愈工作，耕暖生活"那种温润治愈节奏）
- 必查敏感词
- 校验失败：自动改写一次，仍失败则标记"待人工"并通知用户
- 失败时 LLM 自检重试 ≤ 2 次，仍不达标 → 降级提示"内容需人工优化"
- 校验优先级：1.10 全局校验优先级最高，自检失败不算通过

### 5.11 用户偏好与铁律存储（🔴 P0）

**功能目标**：用户对 IP 画像的修改建议沉淀为铁律，永久记忆。

**核心机制**：
- 复用 Phase 1 全局铁律存储
- **触发场景**：
  - IP 画像确认时"改一改"
  - 资产页对某条文案"不满意"
  - 重新生成时"记住我的偏好"
- **铁律示例**：
  - "我的定位词里不要出现'赋能'"
  - "故事要用第一视角"
  - "人设风格选陪伴型，不是专家型"
- **优先级**：用户铁律 > 后台配置规则
- 铁律版本化 + 用户可手动删除单条铁律

### 5.12 全自动暂停 + 紧急制动（🔴 P0）

**功能目标**：IP 生成质量不达标时，自动暂停并提示用户。

**核心机制**：
- 复用 Phase 1 全局组件
- **触发条件**：
  - 调性评分 < 阈值
  - 敏感词命中
  - 字段缺失
  - 用户连续 3 次不满意
- **制动动作**：暂停生成 + 推送提醒 + 建议人工介入

### 5.13 历史版本管理（🟡 P1）

**功能目标**：保留 IP 历次版本，支持查看、对比、切换"启用版本"。

**核心机制**：
- 每次生成新 IP 保留旧版本
- 时间线展示，可查看任意历史版本
- "启用版本"切换后，后续模块以新版本为准
- 历史版本不可删除（只读），可标注"归档"

### 5.14 老师查看学员 IP 档案（🟡 P1）

**功能目标**：老师/合伙人可查看所辖学员的 IP 档案（只读）。

**核心机制**：
- 入口：M6 私域耕耘舱工作台 → 学员详情 → IP 档案 tab
- 权限：只能查看所辖学员（权限隔离）
- 功能：查看 IP 画像 + 加备注（"亮点" / "待提升"）+ 备注沉淀到阶段点评
- 老师查看学员数据需权限校验；后台操作可审计

## 八、非功能性需求

### 8.1 性能

- 对话响应延迟：LLM 推理 < 8s，首字 < 2s（流式输出），单轮 P95 ≤ 4s
- IP 画像生成：< 10s
- 资产生成：名片 < 5s，长简介 < 8s
- 并发能力：同时在线访谈 1000 人（Phase 2 初期）、10000 人（全量）

### 8.2 可用性

- 对话状态实时持久化，中断 0 丢失
- LLM 服务降级：主模型不可用时降级到备选模型，对话仍可继续
- 失败重试：智能追问生成失败时，使用后台预置追问模板兜底
- 跨端持久：用户从 APP 切到 Web，对话进度无缝衔接（基于云端 sessionId）

### 8.3 安全

- 访谈数据加密存储（AES-256）
- 老师查看学员数据需权限校验；后台操作可审计
- 用户数据导出需二次确认 + 验证码
- LLM 调用日志不含完整对话，仅含调用元数据用于成本核算

### 8.4 合规

- 用户授权：进入访谈前明确告知"对话内容将用于 IP 定位生成"
- 数据删除：用户注销账号时，IP 相关数据软删除 + 30 天后硬删除
- 内容审核：资产页内容输出前过敏感词审核
- 用户对话日志保留 90 天用于丰满度迭代，之后脱敏归档
- 老师查看学员对话需用户授权链路完整

### 8.5 兼容性

- 支持移动端（iOS Safari、Android Chrome 微信内置浏览器）
- 支持 PC 端（Chrome、Edge、Safari）
- 支持主流屏幕尺寸（375px / 414px / 768px / 1024px / 1440px）
- Web / 移动端 H5 / 企业微信小程序 三端交互一致

### 8.6 可观测与可配置

- 关键事件埋点：进入 / 每轮 / 中断 / 丰满度达标 / 名片生成 / 用户修改
- 后台话术 / 人设分类 / 丰满度阈值 / 禁用词库 全可后台改

## 十、与全局横切组件的依赖关系

| 横切组件 | 依赖方式 | 接入点 |
|---------|---------|-------|
| **品牌语校验引擎** | 所有 IP 输出（画像、资产） | 生成后 → 校验 → 失败自动改写 |
| **智能统一接入层** | 访谈对话、追问生成、丰满度评分、画像生成 | 所有 LLM 调用入口 |
| **用户铁律存储** | 用户修改建议、重新生成偏好 | 写入 → 加载到 Prompt |
| **后台可视化配置中心** | 对话话术、追问模板、丰满度阈值 | 复用通用配置框架 |
| **全自动暂停 + 紧急制动** | 质量评分阈值触发 | 复用全局组件 |

| **日耕APP积分打通** | 完成 IP 访谈 +10 积分（7天激活 Day1） | 埋点上报 |
| **活码归因追踪** | 不涉及（M1 不产生获客行为） | — |

## 十一、风险与缓解

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| LLM 多轮对话状态机失控 | 🟡 中 | 王昊宇亲自出状态机骨架，不让 AI 自由发挥；上下文超长时做摘要压缩 |
| 抽取准确性不足，关键信息丢失 | 🟡 中 | 信息抽取失败时用规则 fallback；抽取准确率 < 90% 时降级为"引导用户手动补充" |
| 人设分类法扩展性差 | 🟢 低 | 后台预留 8-10 种扩展开关；新增人设无需发版 |
| 对话 token 消耗过高 | 🟡 中 | 上下文压缩 + 摘要；监控单用户成本 |
| 用户卡在对话中出不来 | 🟡 中 | 30 分钟未完成自动保存退出；丰满度阈值降低兜底 |
| 老师权限泄露 | 🟡 中 | 严格 RBAC + 审计日志 + 数据脱敏 |
| 多轮对话 token 累积超上下文窗口 | 🟡 中 | 1.1 中"对话压缩"机制；超 16k 自动摘要 |
| 用户 4 轮即退出，丰满度长期不达标 | 🟡 中 | 1.7 评估器接入；强制收网 + 老师介入 |
| 人设分类命中率低，生成 IP 重复"通用" | 🟡 中 | 1.6 多分类候选 + 用户投票选主用 |
| LLM 调用成本失控 | 🟡 中 | 1.2 用规则评分前置，避免每轮都跑 LLM |
| 品牌语校验与 LLM 自检矛盾（自检说 OK 但全局规则拦截） | 🔴 高 | 1.10 全局校验优先级最高，自检失败不算通过 |
| 老师越权查看学员对话 | 🔴 高 | 8.4 合规链路；后台操作可审计 |
| 用户修改建议进入"铁律"形成循环（建议错但被记住） | 🟡 中 | 铁律版本化 + 用户可手动删除单条铁律 |

## 十二、与 Tech Lead 待对齐议题（王昊宇必看 ⭐）

> 本节列出的 5 个议题，在启动编码前必须由 Tech Lead 给出口径。

### 议题 1：多轮对话 token / 延迟预算

**问题**：单用户从开场到 IP 成形 ~8 轮对话 × 单轮 ~2k token 输入 / ~500 token 输出。TPS 与延迟上限怎么定？

**当前默认假设**（待确认）：
- 单 session 累加 token 上限 16k
- 单轮响应 P95 ≤ 4s
- IP 画像生成 < 10s

### 议题 2：丰满度评分模型

**问题**：V1.0 用规则 + LLM 双轨，是直接上自研小模型还是沿用大模型 LLM-as-judge？

**当前默认假设**（待确认）：
- 短期：规则评分 + LLM-as-judge 双轨，置信度 ≥ 0.8 才允许收网
- 中期：积累 100 样本后，训练领域小模型替代 LLM 评分以控成本

### 议题 3：对话日志存储

**问题**：本地 vs 云端；冷热分层策略。

**当前默认假设**（待确认）：
- 热数据（最近 30 天）：云端 Redis + DB
- 冷数据（30-90 天）：脱敏后归档到对象存储
- 90 天后：硬删除

### 议题 4：跨端一致性

**问题**：Web / H5 / 小程序 三端的 sessionId 与断点续接机制。

**当前默认假设**（待确认）：
- 统一云端 sessionId，三端共用
- 对话状态实时持久化（Redis + DB 双写）
- 退出后 30 天内可继续

### 议题 5：LLM 选型与限流

**问题**：选哪家模型，限流兜底怎么配（参考全局 2.）。

**当前默认假设**（待确认）：
- 主模型：中文长上下文表现优 + 角色一致性强的大模型（具体选型待 Tech Lead 拍板）
- 备选模型：主模型不可用时降级
- 限流：按用户 + 全局双维度，配兜底文案

## 十三、验收标准（DoD）

### 13.1 功能验收

- [ ] 1.1：对话可最少持续 4 轮，最多 30 轮；30 轮未达丰满度阈值也允许收网
- [ ] 1.2：丰满度评分与人工盲评相关度 ≥ 0.7（100 样本测试）
- [ ] 1.3：追问不与最近 2 轮重复；连续 3 轮追问同一缺口后自动换缺口
- [ ] 1.4：后台修改问题模板后，5s 内线上生效；版本可回滚
- [ ] 1.5：100 条样本对话，结构化字段抽取准确率 ≥ 85%
- [ ] 1.6：4 类人设分类可演示；一句话定位生成一次通过率 ≥ 70%
- [ ] 1.7：丰满度评估分数与 1.2 一致，不允许两套分数
- [ ] 1.8：4 平台名片全部生成，所有输出过品牌语校验
- [ ] 1.9：（V1.1）4 类故事模板可配置；用户选择后系统填充字段
- [ ] 1.10：禁用词替换 100% 准确；句式校验通过率 ≥ 95%
- [ ] 1.11：用户改一改的建议下次生成时自动遵循
- [ ] 1.12：紧急制动：质量评分低时自动暂停并提醒
- [ ] 1.13：历史版本可查看、可对比、可切换启用版本
- [ ] 1.14：老师可查看所辖学员 IP 档案（权限隔离）

### 13.2 体验验收

- [ ] 新用户 30 分钟内能完成访谈并生成 IP 画像
- [ ] 访谈中断后能继续，信息不丢失
- [ ] 试用用户 7 天完成率 ≥ 80%（内测 50 人样本）
- [ ] 单次对话中断后 24h 内重连对话进度不丢失
- [ ] 用户对 IP 名片的满意度评分 ≥ 4.0（5 分制）
- [ ] Web / H5 / 小程序 三端交互一致
- [ ] 6 大维度丰满度评估准确，追问合理
- [ ] IP 画像包含：人设标签、专业领域、差异化卖点、人设风格、一句话定位
- [ ] 资产生成：4 平台名片 + 3 长度简介 + 故事 + 5 句背书话术

### 13.3 非功能验收

- [ ] 对话响应延迟 < 8s
- [ ] IP 画像生成 < 10s
- [ ] 对话状态持久化测试通过
- [ ] 1000 并发访谈不崩溃
- [ ] 移动端、PC 端兼容性测试通过
- [ ] 安全性测试：数据加密、权限隔离通过

### 13.4 合规验收

- [ ] 用户对话日志 90 天后自动脱敏归档脚本就绪
- [ ] 老师查看学员对话需用户授权链路完整
- [ ] LLM 调用日志不含完整对话，仅含调用元数据用于成本核算

### 13.5 集成验收

- [ ] 与品牌语校验引擎对接通过
- [ ] 与智能统一接入层对接通过
- [ ] 与铁律存储对接通过
- [ ] 与日耕 APP 积分系统对接：完成 IP +10 积分
- [ ] 与 7 天激活任务引擎对接：Day 1 任务完成态同步
