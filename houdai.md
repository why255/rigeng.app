# 日耕管理后台模块规划（以Claude Code生成为基准，补充细化）

基于Claude Code生成的八大模块框架，结合四板块需求洞察文档和20个服务/70+张表的实际架构，进行补充和细化。

---

## 一、用户与权限管理

> 对应Claude生成的第一模块，后端已写、缺前端UI。

### 1.1 用户列表
| 功能 | 说明 | 数据来源 |
|------|------|---------|
| 用户列表展示 | 昵称/手机号/注册时间/VIP等级/最后活跃/所属老师/用户标签 | 用户服务 |
| 用户详情 | 13模块使用数据（脱敏）、存储空间使用量、贡献值余额 | 各服务汇聚 |
| 角色变更 | 普通用户↔VIP↔老师↔运营↔超级管理员 | 用户/权限服务 |
| 用户标签 | 手动打标（"高潜转型""情绪树洞高频""意向客户""沉默用户""校友"） | 运营后台 |

### 1.2 VIP会员管理
| 功能 | 说明 |
|------|------|
| 套餐配置 | 试用期7天/初级VIP 29元/月/中级VIP 99元/月/高级VIP一人一价 |
| 配额管理 | 各等级存储空间、录音时长/月、视频辅导时长/月 |
| 到期管理 | VIP到期提醒、自动降级、续费记录 |

### 1.3 贡献值管理
| 功能 | 说明 |
|------|------|
| 贡献值规则配置 | 获取规则（采纳20点/拒绝5点/分享成功案例100点等）+消耗规则+比率，后台独立配置入口 |
| 贡献值排行榜 | 平台贡献值排行榜（荣誉榜，非财富榜），等级体系青铜→白银→黄金→钻石 |
| 贡献值明细查询 | 按用户查询贡献值获取/消耗记录 |

### 1.4 用户批量操作
| 功能 | 说明 |
|------|------|
| 批量启停 | 批量冻结/解冻用户账号 |
| 批量导入导出 | 用户数据批量导入导出（合规脱敏） |
| 沉默用户运营 | 连续3/5/7天未使用触发记录、运营官微信私信关怀记录 |

## 三、内容与知识库审核

## 十、开发优先级（细化版）

| 优先级 | 模块 | 说明 | 依赖 |
|:------:|------|------|------|
| **P0** | 用户与权限管理 | 后台基础，无权限什么都做不了 | 用户/权限服务 |
| **P0** | 系统运营监控-仪表盘 | 运营第一时间需要看到数据 | 仪表盘服务 |
| **P1** | 教师管理 | 辅导流程依赖老师体系 | 用户/权限服务 |
| **P1** | 安全与危机监控-危机事件 | 产品安全底线，必须优先 | 安全服务+情绪树洞 |
| **P2** | 内容与知识库审核 | 公私智库运营依赖 | 知识库服务 |
| **P2** | 消息推送管理 | 正反向关怀依赖 | 消息推送服务 |
| **P3** | 系统配置 | 超管配置需求 | 全部服务 |
| **P3** | 审计日志 | 合规需求 | 安全服务 |
| **P3** | 系统运营监控-完整版 | 存储/AI消耗/服务健康 | 各服务 |
